Everything is a Freaking DNS problem - jbilling http://127.0.0.1:8080/blog/taxonomy/term/1195/0 en Open Source does not mean Customization Heaven.. http://127.0.0.1:8080/blog/open-source-does-not-mean-customization-heaven <p>Unless you are doing it wrong.</p> <p>And sadly I`m seeing more and more people doing it wrong.<br /> To a lot of people Open Source means that they have a piece of software that does almost what they want and which they can modify to their best wishes and use internally.</p> <p>So they fork locally,, they don't redistribute their code , but they aren't contributing their changes back upstream, chances are these changes wouldn't be accepted upstream anyhow as they are really customizing the code for their specific cases. At first sight this doesn't look so bad , at second sight ..</p> <p>When weeks or months later the upstream project releases an urgent security fix, the local fork has deviated soo much that it can't upgrade anymore and stays with an insecure version.<br /> Often it's worse.. a feature that could have been accepted upstream has been implemented slightly different in the local fork, the result being that newer features depending on the first one also can't be integrated anymore </p> <p>Some projects are prepared for local contributions, they have a modular framework that allows you to build on top of the project while not having to touch the core of a project, Drupal and openQRM are great examples of those, but not all projects are that smart. Needless to say that when you have such a modular framework you really shouldn't be modifying the core part of the platform, unless you are fixing a real bug.</p> <p>But the general rule of thumb is that when you fix bugs, make sure they are inserted upstream , or implement new features.</p> <p>Now sometimes there is no easy way to get your code accepted upstream, in which case you should announce clearly that you want to contribute but you are blocked and publish the patches somewhere else ... </p> <p>Don't let the community work for you, but work with the community !</p> http://127.0.0.1:8080/blog/open-source-does-not-mean-customization-heaven#comments drupal jbilling mysql open source openqrm sugarcrm sun Wed, 11 Feb 2009 16:09:52 +0000 Kris Buytaert 878 at http://127.0.0.1:8080/blog
