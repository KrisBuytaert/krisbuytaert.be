If however you have a vlan with 3 digits on a bond interface Heartbeat will add :0 to bond0.129 , the Heartbeat resource will add the ip address perfectly but opon checking all the :0 interfaces the bond0.129:0 interface won't be parsed as ifconfig outputs it as bond0.129 , hence resulting in a potential painfull situation where 2 nodes still share an IP address.</p> <p>So where's the actual problem ifconfig, or heartbeat, I'd say both, but the easiest fix will be in Heartbeat, afterall there are other preferred ways of adding an ip addres to an interface. ip addr add comes to mind :)</p> <p>So I filed a <a href="http://developerbugs.linux-foundation.org/show_bug.cgi?id=1968" rel="nofollow">bug report</a> :)</p> http://127.0.0.1:8080/blog/bug-ifconfig#comments ha heartbeat ifconfig ip linux-ha opensource Wed, 24 Sep 2008 15:27:26 +0000 Kris Buytaert 730 at http://127.0.0.1:8080/blog
